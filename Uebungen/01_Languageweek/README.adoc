== Domain Model

Das Modell bildet eine vereinfachte Verwaltung von Sprachwochen ab.
Eine Sprachwoche (_LanguageWeek_) wird für eine bestimmte Klasse (_Schoolclass_) angelegt.
Es gibt immer einen Lehrer, der die Veranstaltung leitet (_Languageweek.Teacher_).
Optional kann ein Begleitlehrer mitfahren (_Languageweek.SupportTeacher_).
Die Ziele (_Destination_) sind zentral in der Datenbank vorgegeben.
Schülerinnen und Schüler können sich zur Sprachwoche anmelden.
Für eine Anmeldung wird eine _Registration_ erstellt, die das Anmeldedatum beinhaltet.

== Aufgabe 1: Service

In _Languageweek.Application/Services/LanguageweekService.cs_ sollen folgende Methoden implementiert werden:

`List<Teacher> GetTeachersWithMinCountOfParticipations(int count)`::
Diese Methode soll ermitteln, welche Lehrer mindestens _count_ mal bei einer Sprachwoche als Leiter oder Begleiter eingesetzt wurden.
Tipp: Verwenden Sie die Navigations _LanguageweekTeachers_ und _LanguageweekSupportTeachers_.
Vergleichen Sie mit `>= count`.

`List<Schoolclass> GetClassesWithoutLanguageWeek()`::
Diese Methode soll ermitteln, welche Klassen _nie_ für eine Sprachreise verplant wurden.

`List<SchoolclassStatistics> CalcSchoolclassStatistics()`::
Diese Methode soll ermitteln, wie viele männliche und weibliche Schüler es pro Klasse gibt.
Als Rückgabe steht Ihnen folgender Record zur Verfügung:
+
[source,csharp]
----
public record SchoolclassStatistics(
    int Id, string Shortname, int MaleCount, int FemaleCount);
----

`List<LanguageWeekRegistrationRate> CalcRegistrationRates()`::
Diese Methode soll eine Statistik für jede Sprachwoche berechnen.
Als Rückgabe steht Ihnen folgender Record zur Verfügung:
+
[source,csharp]
----
public record LanguageWeekRegistrationRate(
    int Id, DateOnly From, DateOnly To, decimal TotalPrice,
    int DestinationId, string DestinationCity, string DestinationCountry,
    int SchoolclassId, string SchoolclassShortname, decimal Percentage);
----
+
Das Property _TotalPrice_ berechnet sich aus der Teilnehmeranzahl (Anzahl der _Registrations_ der Sprachwoche) mal _PricePerPerson_.
Das Property _Percentage_ berechnet sich aus der Teilnehmeranzahl in Relation zur Gesamtzahl der SchülerInnen der betreffenden Klasse.
Runden Sie diesen Prozentwert auf eine Dezimalstelle mit _Math.Round()_.

`Task<LanguageWeek> CreateLanguageWeekAsync(CreateLanguageWeekCommand command)`::
Diese Methode soll eine Sprachwoche anlegen.
Als Command-Objekt dient folgender Record:
+
[source,csharp]
----
public record CreateLanguageWeekCommand(
    int SchoolclassId, int DestinationId, DateOnly From, DateOnly To,
    int TeacherId, decimal PricePerPerson, int? SupportTeacherId);
----
+
Folgende Prüfungen sollen in der Businesslogik vorgenommen werden:
+
*(1)* Wird die ID der Schulklasse nicht gefunden, ist _throw new LanguageweekServiceException("Schoolclass not found.")_ zu werfen. +
*(2)* Wird die ID des Reisezieles (Destination) nicht gefunden, ist _throw new LanguageweekServiceException("Destination not found.")_ zu werfen. +
*(3)* Wird die ID des Lehrers nicht gefunden, ist _throw new LanguageweekServiceException("Teacher not found.")_ zu werfen. +
*(4)* Eine Klasse darf nur einmal auf Sprachreise fahren. Ist die Klasse schon in _Languageweeks_ vorhanden, ist _throw new LanguageweekServiceException("The class already has a language week planned.")_ zu werfen. +
*(5)* Befinden sich männliche Schüler in der Klasse, aber es wurden ausschließlich weibliche Lehrer (Teacher und ggf. SupportTeacher) zugewiesen, werfen Sie _throw new LanguageweekServiceException("The class has male students, but only female teachers are assigned.")_. +
*(6)* Befinden sich weibliche Schülerinnen in der Klasse, aber es wurden ausschließlich männliche Lehrer zugewiesen, werfen Sie _throw new LanguageweekServiceException("The class has female students, but only male teachers are assigned.")_. +
+
Werden alle Bedingungen erfüllt, ist der Datensatz in der Datenbank zu speichern und zurückzugeben.

`Task UpdateLanguageWeekAsync(UpdateLanguageWeekCommand command)`::
Diese Methode soll eine Sprachwoche aktualisieren.
Als Command-Objekt dient folgender Record:
+
[source,csharp]
----
public record UpdateLanguageWeekCommand(
    int Id, int DestinationId, DateOnly From, DateOnly To,
    int TeacherId, decimal PricePerPerson, int? SupportTeacherId);
----
+
Die Klasse darf nicht verändert werden, weshalb sie nicht in _UpdateLanguageWeekCommand_ enthalten ist. Wird ein `SupportTeacher` übergeben (nicht null), soll dieser aktualisiert werden.
+
Folgende Prüfungen sollen in der Businesslogik vorgenommen werden:
+
*(1)* Wird die ID der Sprachwoche nicht gefunden, ist _throw new LanguageweekServiceNotFoundException("Language week not found.")_ zu werfen. +
*(2)* Die Parameter wie Lehrer, Destination und Geschlechter-Matching werden wie in _CreateLanguageWeekAsync_ geprüft.
+
Werden alle Bedingungen erfüllt, ist der Datensatz in der Datenbank zu aktualisieren.

`Task UpdateLanguageWeekPriceAsync(UpdateLanguageWeekPriceCommand command)`::
Diese Methode soll den Preis einer Sprachwoche aktualisieren.
Als Command-Objekt dient folgender Record:
+
[source,csharp]
----
public record UpdateLanguageWeekPriceCommand(int Id, decimal PricePerPerson);
----
+
Wird die Sprachwoche nicht gefunden, ist _throw new LanguageweekServiceNotFoundException("Language week not found.")_ zu werfen.

`Task DeleteLanguageWeekAsync(int id, bool forceDelete = false)`::
Diese Methode soll eine Sprachwoche löschen.
+
Folgende Prüfungen sollen in der Businesslogik vorgenommen werden:
+
*(1)* Wird die ID der Sprachwoche nicht gefunden, ist _throw new LanguageweekServiceNotFoundException("Language week not found.")_ zu werfen. +
*(2)* Vergangene Sprachwochen dürfen nicht gelöscht werden, auch nicht mit _forceDelete_.
      Werfen Sie in diesem Fall _throw new LanguageweekServiceException("Past language weeks cannot be deleted.")_.
      Beziehen Sie die Systemzeit über den bereitgestellten _TimeProvider_ mit _timeProvider.GetUtcNow().DateTime_. +
*(3)* Wenn Schüler angemeldet sind (Daten in _Registrations_ vorhanden sind) und der Parameter _forceDelete_ auf _false_ gesetzt ist, ist
      _throw new LanguageweekServiceException("The language week cannot be deleted because students are already registered.")_ zu werfen.
+
Werden alle Bedingungen erfüllt, ist der Datensatz in der Datenbank zu löschen. Gegebenenfalls müssen, wenn _forceDelete_ auf true gesetzt ist, vorher alle Anmeldungen (_Registrations_) gelöscht werden.

== Aufgabe 2: Controller

In _Languageweek.Api/Controllers/RegistrationsController.cs_ sollen Endpunkte für Anmeldungen (Registrations) implementiert werden.
Der Controller besitzt eine Dependency über das Interface _IRegistrationService_ zum Service.
Der Service _RegistrationService_ ist bereits fertig implementiert und kann verwendet werden.
*Der Controller darf nicht direkt auf den Datenbankkontext zugreifen.*

Als zentrales DTO für die Endpunkte steht Ihnen folgender Record zur Verfügung:

[source,csharp]
----
public record RegistrationDetailDto(
    int Id, DateTime RegisterDate, int StudentId, string StudentFirstname,
    string StudentLastname, string StudentEmail, string SchoolclassShortname,
    int LanguageweekId, string DestinationCity, DateOnly LanguageweekFrom,
    DateOnly LanguageweekTo, string TeacherShortname);
----

`GET api/registrations?schoolclass={name}`::
Dieser Endpunkt liefert eine Liste aller Anmeldungen einer bestimmten Schulklasse als _RegistrationDetailDto_ zurück.
Die Daten können über das Property _Registrations_ des Services abgefragt werden.
Danach wird eine Liste von _RegistrationDetailDto_-Objekten mit HTTP 200 (OK) zurückgegeben.

`GET api/registrations/{id}`::
Dieser Endpunkt liefert Details zu einer Anmeldung mit der übergebenen ID zurück.
+
Wird die ID nicht in der Datenbank gefunden, ist HTTP 404 (Not Found) samt Problem-Detail-Body mit der Meldung _Registration with ID {id} not found._ zurückzugeben.
Im Erfolgsfall ist das DTO mit HTTP 200 (OK) zurückzugeben.

`POST api/registrations`::
Legt eine neue Anmeldung über die Servicemethode _CreateRegistration_ an.
+
Als Command-Objekt steht Ihnen folgender Record zur Verfügung:
+
[source,csharp]
----
public record CreateRegistrationCmd(int LanguageweekId, int StudentId, DateTime RegisterDate);
----
+
Im Erfolgsfall soll HTTP 201 (Created) geliefert werden.
Der Location-Header soll auf die URL der neu angelegten Ressource zeigen (verwenden Sie dafür _CreatedAtAction_).
+
Fangen Sie die _RegistrationServiceException_ ab und liefern Sie HTTP 400 (Bad Request) samt Problem-Detail-Body mit der Message der Exception.
+
Für _CreateRegistrationCmd_ gelten folgende Validierungsregeln.
Implementieren Sie diese über Annotations bzw. die Implementierung des _IValidatableObject_-Interfaces in _CreateRegistrationCmd_:
+
*(1)* LanguageweekId muss mindestens 1 sein. +
*(2)* StudentId muss mindestens 1 sein. +
*(3)* Registrierungen, deren Wert von _RegisterDate_ in der Vergangenheit liegt, sind ungültig.
Verwenden Sie nicht DateTime, sondern holen Sie sich für diese Prüfung mit _validationContext.GetRequiredService<TimeProvider>()_ den Time Provider!

`DELETE api/registrations/{id}`::
Löscht eine Anmeldung über die Servicemethode _DeleteRegistration_.
Im Erfolgsfall soll HTTP 204 (No Content) geliefert werden.
+
Fangen Sie die _RegistrationServiceNotFoundException_ ab und liefern Sie HTTP 404 (Not Found) samt Problem-Detail-Body mit der Message der Exception.
Fangen Sie die _RegistrationServiceException_ ab und liefern Sie HTTP 400 (Bad Request) samt Problem-Detail-Body mit der Message der Exception.

== Aufgabe 3: Integration Tests

Für eine automatisierte Testung sind in der Klasse _Languageweek/Languageweek.Test/RegistrationsControllerTests.cs_ Integration Tests zu schreiben, die folgende Fälle abdecken:

* Der Endpunkt _POST api/registrations_ liefert bei korrekten Daten HTTP 201 Created und trägt den Datensatz in die Datenbank ein.
  Verwenden Sie die Methoden _GetHttpContent_ und _QueryDatabase_ aus der _TestWebApplicationFactory_.
* Der Endpunkt _POST api/registrations_ liefert bei ungültigem _RegisterDate_ HTTP 400 und die Fehlermeldung _Registration date is in the past._
  Verwenden Sie die Methoden _PostHttpContent_ und _ContentContainsErrorMessage_ aus der _TestWebApplicationFactory_.

Achten Sie darauf, dass Sie mit der Methode _InitializeDatabase_ aus der _TestWebApplicationFactory_ minimale Testdaten bereitstellen.
Der Time Provider soll einen fixen Zeitwert liefern; verwenden Sie *nicht DateTime.Now* oder ähnliche Abhängigkeiten zur Systemzeit im Test.
